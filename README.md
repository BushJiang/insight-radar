# 智源雷达（InsightRadar）

智源雷达是一个帮你发现优质开源项目的工具。

它的核心思路是：关注 GitHub 上优秀开发者关注了哪些项目，再用 AI 帮你整理、搜索和推荐这些项目。

## 这个应用能做什么

### 采集项目

你可以输入一个 GitHub 用户名，应用会采集这个账号标星过的开源项目。

适合用来关注：

- 技术大牛最近在看什么项目。
- 某个领域开发者收藏了哪些工具。
- 自己以前标星过的项目里有哪些值得重新整理。

### 生成项目简介

应用会读取项目资料，并用 AI 生成更容易理解的项目简介。

你不用逐个打开 GitHub 仓库看 README，也能快速知道一个项目大概是做什么的。

### 搜索项目

采集后的项目会保存下来，你可以在项目库里搜索和筛选。

适合用来快速找到：

- 某个关键词相关的项目。
- 某种编程语言的项目。
- 某个 GitHub 用户标星过的项目。

### 智能推荐

你可以告诉应用自己关注的方向，比如前端、后端、数据库、AI 工具、开发工具等。

应用会从已采集项目中找出更符合你兴趣的项目，并生成推荐理由。

### 语义搜索

除了普通关键词搜索，应用也支持用语义方式搜索项目。

比如你搜索“适合做 AI 应用记忆系统的项目”，不一定要求项目简介里出现完全一样的词，应用也能尝试找出语义相关的项目。

## 使用前需要准备什么

你需要准备这些东西：

1. GitHub Token

   用来读取 GitHub 标星项目。

2. DeepSeek API Key

   用来生成项目简介、分析项目价值和推荐理由。

3. SiliconFlow API Key

   用来生成项目向量，支持语义搜索和智能推荐。

4. PostgreSQL 数据库

   用来保存采集到的项目和应用设置。

5. Milvus 或 Zilliz

   用来保存项目向量，支持语义搜索。

如果你只是本地试用，也可以使用本地 PostgreSQL 和本地 Milvus。

## 安装和启动

### 1. 安装 Bun

如果你还没有安装 Bun，可以先安装 Bun。

安装方式见 Bun 官网：

```text
https://bun.sh
```

### 2. 安装项目依赖

在项目目录中运行：

```bash
bun install
```

### 3. 创建配置文件

复制一份环境变量样本：

```bash
cp .env.example .env
```

然后打开 `.env`，填写你的数据库和向量数据库配置。

业务 API Key 不需要写进 `.env`。GitHub Token、DeepSeek API Key、SiliconFlow API Key 都是在应用启动后的设置页面里填写。

### 4. 初始化数据库

配置好 `.env` 里的数据库连接后，运行：

```bash
bun run db:init
```

这个命令会自动创建应用需要的数据表。

你不需要手动执行多个迁移文件。

### 5. 启动应用

运行：

```bash
bun run dev
```

然后在浏览器打开：

```text
http://localhost:3000
```

## 第一次使用流程

### 1. 打开设置页面

第一次启动后，先进入设置页面，填写：

- GitHub Token
- DeepSeek API Key
- SiliconFlow API Key
- 推荐偏好

保存后，应用会记住这些设置。

设置页只会显示密钥是否已经设置，不会把已经保存的密钥明文显示出来。

### 2. 采集 GitHub 标星项目

进入项目采集页面，填写 GitHub 用户名，然后开始采集。

采集完成后，项目会进入本地项目库。

### 3. 生成项目简介

采集过程中或采集完成后，应用会为项目生成简介。

这些简介会用于后续搜索和推荐。

### 4. 搜索项目

进入项目库页面，可以按关键词、语言、来源用户等条件搜索项目。

### 5. 获取智能推荐

进入智能推荐页面，输入你想找的项目方向，应用会从项目库中找出候选项目，并生成推荐理由。

## 配置说明

### `.env` 配置什么

`.env` 主要配置应用运行所需的基础服务，例如：

- PostgreSQL 数据库地址
- Milvus / Zilliz 地址
- SiliconFlow API 地址
- 项目向量模型名称

### 设置页面配置什么

设置页面配置业务密钥和推荐偏好，例如：

- GitHub Token
- DeepSeek API Key
- SiliconFlow API Key
- 感兴趣的技术领域
- 推荐候选数量和并发参数
- 项目简介提示词
- 项目推荐提示词

## 关于 API Key 保存

当前应用会把你在设置页填写的 API Key 保存到数据库中。

为了使用方便，目前是明文保存。请不要把数据库暴露给不可信的人。

## 常见问题

### 为什么需要先运行 `bun run db:init`？

应用需要数据库保存项目和设置。`bun run db:init` 会帮你创建需要的数据表。

如果不执行这一步，应用打开后可能无法保存项目或设置。

### 为什么需要 Milvus 或 Zilliz？

它们用于保存项目向量，让应用支持语义搜索和智能推荐。

如果没有配置，普通项目采集和基础搜索可能还能使用，但语义搜索和智能推荐会受到影响。

### 保存设置后，为什么输入框里看不到原来的 API Key？

这是正常的。

为了减少密钥泄露风险，设置页只显示“已设置”或“未设置”，不会回显完整 API Key。

如果你想更换某个 API Key，重新输入新的值并保存即可。

## 注意事项

- 不要公开分享自己的 API Key。
- 当前版本更适合个人本地使用。
